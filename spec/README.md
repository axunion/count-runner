# Count Runner 計画書(v2.1)

群衆ランナー系ゲーム「Count Runner」の計画書。AI(または開発者)がこのドキュメント群と実装コードだけを頼りに、次のイテレーションを完了できることを目的とする。

この計画書は**コードを直接書かない**。仕様・データ構造・数式・パラメータ・検証方法を定義し、コードの書き方は実装者の裁量に委ねる。

> **経緯**: v1(プロトタイプ)・v2(ゲート値抽選・コンボ/フィーバー・ボス戦・スコア・レスポンシブ・モジュール分割)は実装完了済み。完了した計画書は削除済みで、記録は git 履歴を正とする。本書は v2.1(操作・ペーシング・ギミック・演出・実アセット)の**現行版**である。

## ゲーム概要

- モバイル縦画面ファーストのゲーム(v2.1 で PC キーボード操作にも対応)。論理幅 360px 固定・論理高さは端末縦横比に応じて 640〜900px の可変。PC では拡大表示 + 左右テーマ装飾パネル。
- フィールドが下方向へ自動スクロールし、プレイヤーはポインター(またはキーボード)でリーダーを左右に動かす。
- 流れてくる「左右 2 択のゲート行」を通過するたびにユニット数が増減する。効果値はスポーン時に抽選され、毎行が固有の計算問題になる。v2.1 では左右セルの幅が非対称になり、一部の行では境界が振動する。
- ゴール距離に到達すると**ボス戦フィナーレ**: 群れをボスの HP と相殺させ、1 体でも残れば Clear。道中でユニット数が 0 になれば GameOver。1 プレイ約 24 秒。
- クリア時はスコア(残存体数 + 最大コンボ)を算出し、ベスト記録(localStorage)と比較して表示する。
- テーマは「Sword, Shield, and Sorcery」— 見習い魔法使いの群れがマナを集め、聖域 SANCTUM を目指す([01-world-theme.md](./01-world-theme.md))。

## v2.1 の目的

1. **PC でも同等に遊べる** — キーボード操作の追加(モバイルファーストは維持)。
2. **1 サイクルを短く気持ちよく** — 約 24 秒へ短縮、序盤をさらに簡単に、徐々に難しく。
3. **ドキドキ感のあるゲート** — 幅の非対称・振動する境界・不等間隔・高速行。
4. **選択の実感** — 選んだ側/選ばなかった側で異なるフィードバック。
5. **世界観の体感** — SVG アセットを制作して差し替え(仕組みは v2 で実装済み)。群れの人数を画像の可読性に合わせて調整。

## 前提スタック(重要)

このリポジトリは **SolidJS** テンプレートである。**React ではない**。

| 項目 | 内容 |
| --- | --- |
| UI フレームワーク | SolidJS ^1.9.13(`createSignal` / `onMount` / `onCleanup` / `<Show>`。React API は存在しない) |
| ビルド | Vite 8 + vite-plugin-solid + Lightning CSS |
| TypeScript | ~6.0.2。`verbatimModuleSyntax`(型 import は `import type` 必須)+ `erasableSyntaxOnly`(**enum / namespace 禁止**) |
| CSS | CSS Modules(`*.module.css`) |
| Lint / Format | Biome(ダブルクオート、未使用変数・引数エラー) |
| テスト | Vitest(jsdom、`pnpm test`) |
| パッケージ管理 | pnpm(Node.js 24) |
| Git hooks | lefthook(pre-commit: biome 自動修正 / pre-push: `pnpm check` + `pnpm test`) |

エントリポイントは `src/index.tsx` → `src/App.tsx`。相対 import には拡張子を付ける。パスエイリアスは導入しない。ゲームライブラリ(PixiJS / Phaser 等)は引き続き導入しない — 描画量は素の Canvas 2D の 60fps 圏内であり、大量パーティクル等が必要になった時点で WebGL 移行を検討する。

## 設計上の最重要原則(v2 から不変)

1. **リアクティブ境界の最小化** — Solid のシグナルに載せるのは HUD / オーバーレイの DOM 表示値のみ。フレームごとに変わる状態は plain object に置き、rAF ループが直接書き換える。**ユニット単位のリアクティブ更新は禁止**。
2. **見た目の完全分離** — テーマ文字列・色・画像参照は `theme/themeConfig.ts` にのみ存在する。描画コードは「テーマに画像があれば画像、なければプレースホルダー図形」を自動で選ぶ。**画像差し替えにコード変更を要求しない**。
3. **面白さ優先の調整可能性** — バランスに関わる数値はすべて `constants.ts` の名前付き定数とし、[02-design-v2.1.md](./02-design-v2.1.md) §7 の「調整ノブ」と対応させる。
4. **ロジックと表示の分離** — シミュレーションは `logic/`(純関数 + `stepWorld`)に閉じ、Solid シグナルへの反映は `Game.tsx` がイベントを写像する形で行う。新しいゲームロジックは純関数として `logic/` に置き、テストを付ける。
5. **品質ゲート** — 各実装フェーズで動作確認を行い、最終的に `pnpm check` / `pnpm test` / `pnpm build` がすべて成功すること。
