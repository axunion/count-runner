# 04. アセット仕様(画像差し替え規約)

ゲームはプレースホルダー図形でも動作するが、世界観の本命はキャラクター・オブジェクト・背景の画像である。v2.1 では SVG アセットを制作して同梱する([05-implementation-plan-v2.1.md](./05-implementation-plan-v2.1.md) Phase 4)。本章は**「制作した画像を、コード変更なしで差し替えられる」**ための規約である。アセット制作者は本章と [01-world-theme.md](./01-world-theme.md) §2(アートディレクション)だけを見て制作できることを目指す。

## 1. 差し替えの仕組み(全体像)

```
制作した画像を規約サイズで書き出す
    ↓
public/assets/themes/fantasy/ に規約名で置く
    ↓
themeConfig.ts にエントリを追記(Canvas 要素は assets、クロームは chrome セクション)
    ↓
pnpm dev で確認 — コンポーネント / render/ / CSS の変更はゼロ
```

- Canvas 描画要素は「アセット定義があり、かつロード成功していれば画像。それ以外はプレースホルダー図形」を要素ごとに自動で選ぶ(`render/` 各関数の image/placeholder 分岐)。
- クローム(ゲーム外周)画像は Canvas を通らず、`chrome` セクションの URL を CSS カスタムプロパティ経由で `background-image` に流す([01](./01-world-theme.md) §3)。未定義なら `chrome.backgroundColor` の 1 色で成立する。
- **部分差し替えが可能**であること: 例えばユニット画像だけ先に完成した場合、ユニットのみ画像・他はプレースホルダーで混在動作する。
- アセットのロード失敗(パス間違い等)はゲームを止めず、プレースホルダーで続行し、コンソールに警告を出す。

## 2. 制作の基本ルール

| 項目 | 規約 |
| --- | --- |
| 解像度基準 | ゲームは論理幅 360px で描画(高さは端末により 640〜900px)。**制作は論理サイズの 2 倍(@2x)のピクセル数**で行う(例: 論理 24×24 の要素 → 48×48px で制作)。PC ではさらに CSS 拡大がかかるが、@2x + 総倍率キャップ 3 の範囲では実用上十分 |
| フォーマット | **SVG または PNG(透過)**。SVG の場合、ルート要素に **@2x ピクセル値と等しい `width` / `height` 属性を必ず持たせる**(`naturalWidth` が不定になるとスプライトシートのコマ割り計算が壊れるため)。viewBox のみの SVG は不可 |
| アンカー(基準点) | **画像の中心**。ゲームは要素の中心座標に画像の中心を合わせて描く |
| 余白 | 意味のない透明余白を残さない(表示サイズ計算が狂うため)。発光などのはみ出し表現は表示サイズに含めて設計する |
| 向き | キャラクターは**上向き(奥へ走る後ろ姿)**が正位置。ゲームの進行方向は画面上方向であるため |
| 背景タイル | 幅は論理 360px 固定、**上下端がシームレス**に繋がること(縦ループスクロール)。論理高さが可変でもタイルの繰り返しで自然に対応できる |

## 3. 必要アセット一覧(サイズ確定表)

### Canvas 描画アセット(theme.assets)

「ID」は theme.assets のキー名(§4)、「論理表示サイズ」がゲーム内で占める大きさ、「制作サイズ」が実際に書き出す px。ファイル名は §6。優先度: ★★★ = 世界観の体感に最も効く、★☆☆ = 無くても成立。

| ID | 用途 | 論理表示サイズ | 制作サイズ(@2x) | 構成 | 優先度 |
| --- | --- | --- | --- | --- | --- |
| `unit` | 見習い魔法使い(群れの 1 体) | 24×24 | 48×48 × 4 フレーム → **192×48 の横 1 列シート** | 走りアニメ 4 フレーム、8fps 目安 | ★★★ |
| `enemyUnit` | 敵集団(Stone Legion)の 1 体 | 24×24 | 192×48(unit と同じ 4 フレームシート) | guard クラスタで共有。省略時は敵色の円 | ★★☆ |
| `leaderGlyph` | リーダー足元の魔法陣 | 32×32 | 64×64 | 1 枚。**回転はコード側**で行うため画像は静止で描く | ★★☆ |
| `gateAdd` | Mana Font(マナの泉)バナー | **可変幅**×56(基準 164×56) | 328×112 | 1 枚。通過済みの減光はコード側の透明度で行うため状態違い不要 | ★★★ |
| `gateMultiply` | Chronos Gate(刻の門)バナー | 同上 | 328×112 | 同上 | ★★★ |
| `gateSubtract` | Gargoyle Wall(敵壁)バナー | 同上 | 328×112 | 同上 | ★★★ |
| `boss` | Gate Guardian(フィナーレのボス) | 120×120 | 240×240 | 1 枚。HP 数値はコード側が描く | ★★☆ |
| `background` | フィールド背景タイル | 360×320(縦ループ) | 720×640 | 上下シームレス。地面 + 荒野の小物 | ★★☆ |
| `goalBanner` | SANCTUM(ゴールの門) | 360×80 | 720×160 | 1 枚 | ★★☆ |
| `overlayClear` | クリア画面の挿絵 | 240×160 | 480×320 | 1 枚(DOM の `<img>` で表示) | ★☆☆ |
| `overlayGameOver` | ゲームオーバー画面の挿絵 | 240×160 | 480×320 | 同上 | ★☆☆ |

サイズの根拠:

- **unit 24×24**: プレースホルダー円(半径 8 = 直径 16)より一回り大きい。ユニットの中心間隔は最小 12px(`FORMATION_SPACING`)で意図的に重なり合い、群体として見える。上限 120 体時の群れ直径 ≈ 263px に対して画面幅 360px に収まる。**1 体 1 体が判別できる大きさ**が v2.1 の可読性リバランス([02-design-v2.1.md](./02-design-v2.1.md) §6)の前提。
- **gate バナー(基準 164×56、可変幅)**: v2.1 のセル幅は境界位置により **76〜252px の可変**([02-design-v2.1.md](./02-design-v2.1.md) §3)。バナーは実セル幅に合わせて横に伸縮描画されるため、**横方向の伸縮に耐える構図**(フレーム + 水平方向のテクスチャ。中央の円形エンブレム等は避ける)にすること。`displayValue`("+7" 等)の文字はコード側が上描きするため、**画像内に効果数字を焼き込まない**こと(値は毎回抽選されるため焼き込みは成立しない)。名称ロゴ(Mana Font 等)は画像に含めてよい。
- **boss 120×120**: 画面上部で群れ(直径 ≈263px)と対峙して見劣りしない大きさ。HP 数値・`hpLabel` はコード側が描くため焼き込まない。
- **background 360×320**: 縦ループタイル。論理高さが 640〜900 のどれでも「必要枚数 + 1 枚」の繰り返しで隙間なく敷ける。1 枚を大きくしすぎない(初回ロード対策)。

### クローム画像(theme.chrome。CSS 配信)

Canvas に描かないため論理サイズの規約はなく、**推奨サイズ + CSS の cover/repeat 前提**で制作する。

| ID | 用途 | 推奨サイズ | 表示方法 |
| --- | --- | --- | --- |
| `chrome-background` | ページ全面の背景 | 1920×1280 以上 | `background-size: cover`(中央基準で切り抜かれる前提の構図にする) |
| `chrome-side-panel` | ワイド画面の左右装飾列 | 480×1440(縦長) | 列に対して cover。左右で同一画像(ミラー表示はしない) |

- クローム画像は**装飾**であり、切り抜かれても破綻しない構図(端に重要要素を置かない)にする。
- ゲーム領域より一段暗いトーンにする([01](./01-world-theme.md) §2)。

## 4. themeConfig.ts の assets セクション(型仕様)

[01-world-theme.md](./01-world-theme.md) §3 の `ThemeAssetConfig` に含まれる `assets` の定義。

**SpriteAsset(1 エントリの型)**

| フィールド | 型 | 必須 | 意味 |
| --- | --- | --- | --- |
| `src` | string | ✓ | 画像パス(`/assets/themes/fantasy/...`。public 配下なのでルート絶対パス) |
| `displayWidth` / `displayHeight` | number | ✓ | 論理表示サイズ(§3 の表の値) |
| `frameCount` | number | — | シートのフレーム数。省略時 1(静止画) |
| `fps` | number | — | アニメ再生速度。省略時 8 |

**ThemeAssets(assets セクションの型)** — 以下のキーをすべて **optional** で持つ:

`unit` / `enemyUnit` / `leaderGlyph` / `gateAdd` / `gateMultiply` / `gateSubtract` / `boss` / `background` / `goalBanner` / `overlayClear` / `overlayGameOver`(すべて SpriteAsset)

- ゲート 3 種のキーは GateKind と機械的に対応付けられる命名にする(描画分岐で `kind` から引けること)。
- **未定義キー = プレースホルダー描画**。これが差し替えの ON/OFF スイッチであり、他のフラグを設けない。
- クローム画像はこの型に**含めない**(`chrome` セクションの素の URL 文字列。[01](./01-world-theme.md) §3)。

## 5. 描画側の対応規則(実装要件)

| 要素 | 画像があるときの描画 |
| --- | --- |
| unit | 全ユニットが同一シートを共有。フレーム番号は `floor((elapsed × fps + 位相) mod frameCount)` で経過時間から算出(位相は `wobblePhase` を流用し、群れが一斉に同じポーズにならないようにする)。1 体 1 回の drawImage。**回転・per-unit の状態保持・save/restore は禁止** |
| enemyUnit | guard クラスタの各体に unit と同じ規則で描く。未定義時は敵色(`theme.enemy.color`)の円クラスタ |
| leaderGlyph | リーダー座標を中心に `elapsed × GLYPH_ROT_SPEED` の回転をかけて描く(save/translate/rotate/restore はフレーム 1 回のみなので許容)。フィーバー中の発光表現(色・拡大)は画像にも適用する |
| gateAdd / gateMultiply / gateSubtract | **実セル矩形**(境界位置による可変幅)にバナーを伸縮して描き、その上にコードで `cell.displayValue` 文字を重ねる。解決済みセルの透明度・ハイライトは選択フィードバック([02-design-v2.1.md](./02-design-v2.1.md) §4)がコード側で行う |
| boss | `BOSS_Y` を中心にボス画像を描き、その下にコードで `hpLabel` + HP 数値を重ねる |
| background | タイルを `distance mod タイル高さ` のオフセットで縦に並べてループ描画(枚数は `ceil(viewH / タイル高さ) + 1`)。縞・分割線のプレースホルダー描画は行わない |
| goalBanner | ゴールライン位置にバナーを描く。プレースホルダーの線 + ラベルは行わない |
| overlayClear / overlayGameOver | Canvas ではなく DOM オーバーレイ内の画像として表示(定義が無ければ画像なしの文字のみ) |
| chrome-* | Canvas を通らない。`chrome` セクションの URL を CSS 変数として注入。ロード失敗時はブラウザが背景画像を描かないだけで、地色(`chrome.backgroundColor`)が残る(専用のフォールバック処理は不要) |

**プリロード**: ゲーム開始時(マウント時)に assets の定義済みエントリをすべて画像ロードし、ロード完了を要素ごとに記録する(`theme/assetLoader.ts`)。ロード完了までの間および失敗時はプレースホルダーで描画し続ける(ロード待ちでゲームを止めない)。

## 6. ファイル配置と命名

```
public/
  assets/
    themes/
      fantasy/
        unit.svg              (192×48, 4 フレーム)
        enemy-unit.svg        (192×48, 4 フレーム)
        leader-glyph.svg      (64×64)
        gate-add.svg          (328×112)
        gate-multiply.svg     (328×112)
        gate-subtract.svg     (328×112)
        boss.svg              (240×240)
        background.svg        (720×640)
        goal-banner.svg       (720×160)
        overlay-clear.svg     (480×320, v2.1 では未使用)
        overlay-gameover.svg  (480×320, v2.1 では未使用)
        chrome-background.svg (1920×1280 以上)
        chrome-side-panel.svg (480×1440)
```

- ファイル名は assets のキーのケバブケース(例外: `overlayGameOver` → `overlay-gameover.svg`。gameover は 1 語扱い)。テーマ追加時は `themes/<テーマ名>/` ディレクトリを増やす。拡張子は SVG / PNG のどちらでもよい(§2)。
- このディレクトリは v2.1 Phase 4([05-implementation-plan-v2.1.md](./05-implementation-plan-v2.1.md))で実アセットとともに作成する。

## 7. 差し替え検証(受け入れ基準)

差し替え機構の受け入れ基準は [05-implementation-plan-v2.1.md](./05-implementation-plan-v2.1.md) Phase 5 に統合した(実アセットで実施):

- 部分差し替えが機能する(任意のエントリを外すとその要素だけプレースホルダーに戻る)。
- ロード失敗(パス間違い)時、コンソール警告を出しつつプレースホルダーで続行する。
- アセットの追加・削除で触るソースファイルが `themeConfig.ts` のみである。
