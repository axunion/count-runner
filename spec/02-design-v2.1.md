# 02. デザイン v2.1(操作・ペーシング・ギミック・演出)

v2(ゲート値抽選・コンボ/フィーバー・ボス戦・スコア)の実装完了を受けた、次のイテレーションのデザイン文書。プレイフィールの判断に迷ったらこの章を判断基準にする。定数の実値は `src/game/constants.ts` を正とする(v2 で 03-architecture.md にあった定数表はコードに一本化した)。

## 0. v2 の診断(v2.1 が解決する課題)

| 課題 | v2 の症状 | v2.1 の対策 |
| --- | --- | --- |
| **PC で操作できない** | 入力がポインターのみ。PC ではドラッグ操作が窮屈 | キーボード操作(§5)。モバイルファーストは維持 |
| **1 周が長く、序盤から難しい** | 約 21 行・30〜35 秒。Phase 1 から値比較を要求 | 全体を約 16 行・24 秒に短縮し、序盤をさらに簡単に(§2) |
| **リズムと形が単調** | 行間隔が完全に等間隔、左右セルが常に同幅・同サイズ | 不等間隔・幅の非対称・振動する境界・高速行(§3) |
| **選択の実感が薄い** | 通過時に左右セルが同時に同じだけ減光し、どちらを選んだか分からない | 選択側/非選択側で異なるフィードバック(§4) |
| **見た目がプレースホルダー** | 全要素が単色図形。世界観の楽しさが伝わらない | SVG アセットを制作して差し替え([04-assets.md](./04-assets.md))+ 群れの可読性リバランス(§6) |

## 1. 引き継ぐ設計原則(v2 の面白さの柱)

v2 で確立した以下の柱は不変。これを損なう実装・調整はしない。

- **A. 増殖の快感** — 群れが「面積」で成長を感じさせる。ひまわり配置、×2/×3 の爆発。
- **B. 判断の緊張と手応え** — 毎行が固有の計算問題(値抽選)。「明らかな正解」と「所持数依存の正解」が混ざり続ける。
- **C. 喪失のリスク** — 0 になったら終わり。danger 行・guarded 行の緊張。
- **D. クライマックスと消費** — 増やした群れをボス戦で使い切る。
- **E. 記録と上達の可視化** — スコア + ベスト記録 + NEW RECORD。

失敗の納得感の技術要件も維持する: 隠し情報なし(ゲート値・敵集団数・ボス HP は常時表示)、dt クランプ、リーダー位置基準の一意な判定。**ゲートの視認猶予は最短でも 2.0 秒を下回らない**(例外は §3.4 の高速行のみ。例外条件を明記する)。

## 2. ペーシング再設計

**狙い**: 「もう 1 回」がさらに軽い約 24 秒へ短縮し、序盤は操作を覚えるだけで気持ちよく進めるようにする。行数を減らすぶん、1 行あたりの密度(ギミック)で単調さを消す。

| 項目 | v2 | v2.1 | 意図 |
| --- | --- | --- | --- |
| ゴール距離 `GOAL_DISTANCE` | 6000 | **4600** | 道中 ~22 秒 + フィナーレ ≈ 計 24 秒 |
| フェーズ境界 `PHASE2_START` / `PHASE3_START` | 2000 / 4200 | **1400 / 3000** | 学習 ~5 行 → 最適化 ~6 行 → 試練 ~5 行 ≈ 計 16 行 |
| スクロール速度 `SCROLL_SPEED_BASE` → `MAX` | 180 → 230 | **170 → 250** | 序盤をゆるく、終盤の時間圧を強く |
| Phase 1 の行パターン率 | 加算60/乗算25/乗加15 | **加算70/乗算20/乗加10** | 序盤は「見て、避けて、取る」に集中させる |
| ゲート値レンジ | 下表 | 下表 | 上限 120 体(§6)に合わせて縮小。序盤はさらに狭く |
| 行間隔 `ROW_INTERVAL` | 260 固定 | **廃止** → §3.3 の可変間隔 | |

**ゲート値レンジ(v2.1)**:

| kind | Phase 1 | Phase 2 | Phase 3 |
| --- | --- | --- | --- |
| add | +2〜+4 | +3〜+6 | +4〜+8 |
| multiply | ×2 のみ | ×2(90%)/ ×3(10%) | ×2(85%)/ ×3(15%) |
| subtract | -1〜-2 | -2〜-4 | -3〜-6 |

Phase 2 以降の行パターン率(乗算 vs 加算 30/25%、danger 10/20%、guarded 10/20%)は v2 から変更しない。GOAL_SAFETY(400)も維持し、ボス戦前の静けさを残す。

**期待値の検算**: 最善プレイで Phase 2 中盤に上限 120 へ到達 → Phase 3 の削りを吸収して 90〜120 体でボス戦(HP 60)へ。中間プレイは 40〜80 体で僅差の削り合い。悪い側を引き続ければ初期 10 体は Phase 1 でも全滅し得る(緊張は最初からある)。

## 3. ゲートのギミック(v2.1 の核)

行の形とリズムに変化を与える 4 種。**すべて情報は事前に画面表示され、隠し要素はない**(納得感の原則)。

### 3.1 幅の非対称(リスク・リワード)

- 各行の左右セルは中央固定分割(x=180)ではなく、**境界 `boundaryX` を持つ**。Phase 2 以降、境界は**良い側(kind ランク multiply > add > subtract。danger 行は減少値が小さい側)へ `GATE_ASYM_OFFSET`(50)シフト**する — つまり**良いゲートほど狭い**。
- 狙って入る操作の意味が生まれ、等幅の単調さが消える。狭い側は視覚的に「当たり」のシグナルにもなる。
- **意図的な例外**: 「狭い = 良い」は kind ランクで決めるため、所持数依存では悪い側が狭くなるケースが稀にある(例: 所持 3 体で ×2 vs +4)。幅読みが必勝法にならないスパイスとして許容する(仕様であり不具合ではない)。
- 境界は `GATE_BOUNDARY_MIN`(92)〜`GATE_BOUNDARY_MAX`(268)にクランプし、**最小セル幅 76px を保証**する(必ず入れる。リーダーのクランプ余白より広い)。

### 3.2 振動する境界(ランダムな一部の行)

- Phase 2 以降の一部の行(`GATE_OSC_RATES`: Phase 2 で 15%、Phase 3 で 30%)は、境界が **sin 波で左右に振動**する(振幅 25〜45px、周期 1.6〜2.4 秒。スポーン時に抽選)。
- 境界の最大移動速度は約 177px/s で、リーダーの追従・キーボード移動(420px/s)より遅い — **見て追えば必ず勝てる**が、直前まで迷うと押し出される。ドキドキ感の主源。
- 判定は**通過フレーム時点の境界位置**で行い、解決した瞬間に境界を凍結する(選択ハイライトが動かない)。判定と描画は同一の関数(`boundaryXAt`)を使い、**見えている境界と判定が常に一致**する。
- Phase 1 では出さない(学習の妨げになるため)。

### 3.3 不等間隔スポーン

- 行間隔は固定 260 を廃止し、**フェーズ別基本間隔**(`ROW_GAP_BASE`: 300 / 250 / 220)**± ジッター**(`ROW_GAP_JITTER` 50)とする。序盤はゆったり、終盤は詰まる。
- さらに Phase 2 以降、低確率で**クラスタ**(`ROW_CLUSTER_RATE` 15%: 間隔 150 で連続)と**ブリーザー**(`ROW_BREATHER_RATE` 15%: 間隔 1.5 倍の休符)をロールし、リズムに緩急を作る。

### 3.4 高速行(Phase 3 限定)

- Phase 3 の一部の行(`FAST_ROW_RATE` 20%)は、スクロールに対して `FAST_ROW_SPEED_MULT`(1.4 倍)で**流れてくる速度が速い**。
- 視認猶予は約 1.8 秒と、2.0 秒の下限をわずかに下回る**明示的な例外**。緩和条件: (1) Phase 3 限定、(2) 行の上部にストリーク線の**予告演出**を必ず付ける、(3) スポーン間隔を 1.4 倍(`FAST_ROW_EXTRA_GAP_MULT`)確保して前の行に追突させない。
- プレイテストで理不尽と感じたら倍率を 1.25 へ下げる(§7)。

## 4. 選択フィードバック

通過時に「どちらを選んだか」が一目で分かるようにする。行は `chosenSide` と `resolvedAt` を記録し、純関数 `resolveFeedback(chosen, age)` が透明度を決める。

| 対象 | 挙動 |
| --- | --- |
| **選択した側** | alpha 0.95 を `RESOLVE_CHOSEN_HOLD`(0.35 秒)保持してから 0.4 へイーズ。さらに**ハイライトリング**(セル外周の白系ストローク)が `RESOLVE_RING_LIFETIME`(0.4 秒)で減衰しながら光る |
| **選択しなかった側** | 0.85 → 0.1 へ `RESOLVE_UNCHOSEN_FADE`(0.25 秒)で高速フェード。「消えた方が選ばれなかった方」と直感的に読める |

- リングは塗り(またはスプライト)の**後**に描く。透明度はスプライトにも同一に適用されるため、画像差し替え後も追加分岐なしで成立する。
- フローティングテキスト(選択セル位置に "+7" 等)は v2 から継続。

## 5. キーボード入力(PC)

モバイルファースト(ポインター操作)は不変のまま、PC でも同等に遊べるようにする。

| キー | 動作 |
| --- | --- |
| ← / → 、A / D | 押している間、リーダーの目標位置 `targetX` を `KEYBOARD_MOVE_SPEED`(420 論理px/s)で移動。既存の追従補間がそのまま平滑化を担う |
| Enter / Space | リザルトオーバーレイ表示中のみ Retry |

- **スナップ式(左右レーンへ即ジャンプ)にはしない**。振動する境界(§3.2)を狙って止まる操作スキルを残すため、ドラッグと同じ「位置を運ぶ」操作に揃える。
- **競合規則**: ポインターがアクティブな間はポインターが絶対優先(キー入力は無視)。ポインターを離せば押しっぱなしのキーが次フレームから効く。
- 左右同時押しは移動 0。ウィンドウの blur でキー状態をクリアする(Alt+Tab で押しっぱなしにならない)。矢印キーと Space はページスクロールを抑止する。

## 6. 群れの可読性リバランス

SVG キャラクター([04-assets.md](./04-assets.md))を 1 体 1 体判別できるようにするため、**人数側を調整する**(キャラを小さくするのではなく)。

| 定数 | v2 | v2.1 | 根拠 |
| --- | --- | --- | --- |
| `MAX_UNITS` | 300 | **120** | 表示サイズ 24px のキャラが個体として読める上限。120 体でも群れ直径 ≈ 263px は画面幅 360px を圧倒する(柱 A は維持) |
| `UNIT_RADIUS` | 5 | **8** | プレースホルダー円をスプライト footprint(24×24)に一致させる |
| `FORMATION_SPACING` | 7 | **12** | 重なり比率を v2 と同等に保つ(12/24 ≈ 7/16) |
| `BOSS_HP_BASE` | 100 | **60** | 中間プレイの期待所持数帯が 40〜80 に下がるため |
| `BATTLE_DRAIN_RATE` | 40 | **30** | フィナーレの決着が 1.5〜3 秒に収まる速度 |
| `BEST_STORAGE_KEY` | `count-runner:best` | **`count-runner:best-v2.1`** | 旧上限 300 時代のベストが実質更新不可能になるため、キーを切り替えて仕切り直す |

ゲート値レンジの縮小(§2)もこのリバランスの一部である。

## 7. 調整ノブ(バランス調整ガイド)

プレイテスト後に「どう感じたら、どの定数を触るか」。定数の実値は `src/game/constants.ts`。

| 感じた問題 | 触るノブ | 方向 |
| --- | --- | --- |
| 忙しすぎる / 退屈 | `SCROLL_SPEED_BASE/MAX`、`ROW_GAP_BASE` | 速度を下げる / 間隔を広げる。視認猶予 2.0 秒の下限は死守 |
| 1 プレイが長い・短い | `GOAL_DISTANCE` | 4600 ± 1000 の範囲で |
| 序盤で死にすぎる | `INITIAL_UNITS`、Phase 1 の subtract レンジ | 初期数を増やす / レンジを狭める |
| 終盤に緊張がない | Phase 3 の danger / guarded 率、`FAST_ROW_RATE` | 増やす(danger + guarded 合計 50% 上限目安) |
| 判断が簡単すぎる / 難しすぎる | 値レンジ、`GATE_X3_RATES` | レンジを広げると分岐点の計算が増える。Phase 1 は狭いまま維持 |
| 幅読みだけで解けてしまう | `GATE_ASYM_OFFSET` | 下げると幅のシグナルが弱まる。0 で v2 の等幅に戻る |
| 振動が理不尽 / 空気 | `GATE_OSC_AMP_*`、`GATE_OSC_PERIOD_*`、`GATE_OSC_RATES` | 境界速度がリーダー移動より速くならない範囲で |
| 高速行が理不尽 | `FAST_ROW_SPEED_MULT` | 1.4 → 1.25。それでも駄目なら `FAST_ROW_RATE` を 0 に |
| リズムが単調 / 落ち着かない | `ROW_GAP_JITTER`、`ROW_CLUSTER_RATE`、`ROW_BREATHER_RATE` | ジッターと緩急の量を増減 |
| guarded を誰も選ばない / 選び得 | `GUARD_RATIO_MIN/MAX` | 0.3〜0.6 が駆け引き圏 |
| ボス戦があっけない / 理不尽 | `BOSS_HP_BASE`、`BATTLE_DRAIN_RATE` | HP は中間プレイの期待所持数帯(40〜80)の上限寄り |
| フィーバーが空気 / 強すぎ | `COMBO_FEVER_THRESHOLD`、`FEVER_DURATION`、`FEVER_GAIN_MULT` | 1.5 倍 → 2 倍は劇薬 |
| 増殖の快感が弱い | `GATE_X3_RATES`、`MAX_UNITS` | ×3 の頻度を上げる。MAX を上げる場合はキャラ可読性(§6)とのトレードオフを確認 |
| 操作が重い / 軽すぎる(ポインター) | `LEADER_LERP_RATE` | 8(重い)〜16(ピタ付き) |
| キーボードが遅い / 過敏 | `KEYBOARD_MOVE_SPEED` | 340〜500。振動境界(最大 ~177px/s)より必ず速く保つ |
| 群れがだらしなく見える | `UNIT_FOLLOW_RATE`、`FORMATION_SPACING` | 追従を上げる / 間隔を詰める |
| 選択フィードバックが見えない | `RESOLVE_CHOSEN_HOLD`、`RESOLVE_RING_LIFETIME`、`RESOLVE_UNCHOSEN_FADE` | 保持・寿命を伸ばす |

## 8. プレイテスト自問リスト(自己検証)

実装完了時、実際にプレイして自問し、結果を報告すること。**NO が 3 つ以上あればリリース前に §7 のノブで調整する。**

- [ ] ゲートの**値を読んで比較する瞬間**が毎行あったか(柱 B)
- [ ] **どちらのセルを選んだか**が、通過の瞬間に視覚だけで分かったか(§4)
- [ ] 振動する境界の行で、**直前に判断を変える(あるいは踏みとどまる)瞬間**があったか(§3.2)
- [ ] 狭い側を狙って入るとき、**操作に緊張**があったか(§3.1)
- [ ] キーボードだけで 1 プレイを通し、**ドラッグと同等に操作できた**か(§5)
- [ ] 群れの中の**キャラクター 1 体が一目で判別**できたか(§6 + [04-assets.md](./04-assets.md))
- [ ] ×2/×3 を通過した瞬間、群れの膨張に**気持ちよさ**があったか(柱 A)
- [ ] danger 行・高速行が近づいたとき、**身構える感覚**があったか(柱 C)
- [ ] コンボが伸びているとき、**切りたくない**という意識が生まれたか
- [ ] ボス戦で HP の削り合いに**緊張**があり、勝った瞬間に**達成感**があったか(柱 D)
- [ ] リザルトを見て「**次はもっと**」と思えたか(柱 E)
- [ ] GameOver になったとき、「判断/操作を間違えた自分のせい」と感じられたか(理不尽でないか)
- [ ] 約 24 秒のプレイ中、**待つだけの時間**が 3 秒以上続かなかったか(GOAL_SAFETY 区間とフィナーレを除く)
- [ ] 初見のつもりで見て、各ゲートの良し悪しが**色と形と表記だけで即座に**判別できたか

## 9. v3 候補(今回も見送るもの・見送り理由つき)

| 候補 | 内容 | 見送り理由 |
| --- | --- | --- |
| **Obstacles(回避障害物)** | 行間に静的な柱/壁を置き、接触したユニットだけ死ぬ | 唯一 per-unit 衝突が必要。「末尾切り捨て = 最外周から消える」規律と formation 再割当ての設計を壊す。実装 L |
| **Dynamic Gates(変動値ゲート)** | セル値がスポーン後に周期振動 | v2.1 の**振動する境界**(§3.2)が「動く標的」のファンタジーを低コストで代替した。値の振動は判定表示の一致が崩れやすく fun-per-cost が低いまま |
| **複数ステージ / エンドレス** | パラメータセット化(`StageConfig[]`) | まず 1 ステージの密度を上げるのが先(v2.1 がまさにそれ)。定数は将来括り出せる形(名前付き定数の集約)を維持 |
| **タイトル画面・タップスタート** | `GamePhase` の `"ready"` を使う開始画面 | 型のみの拡張点として維持(`logic/types.ts` に定義済み・未使用) |
| **効果音・BGM・パーティクル・画面シェイク・石化アニメーション・ボス登場カットイン** | 演出強化 | アセット・演出強化フェーズで実施 |
